# Part III — Dialplan & Call Features {.unnumbered}

La Part III est le moment où Asterisk cesse d'être un simple commutateur SIP pour devenir un véritable système téléphonique. Le dialplan est le cœur de cette transformation : c'est la logique qui décide du sort de chaque appel. Nous l'abordons depuis ses principes fondamentaux jusqu'aux modèles avancés — correspondance de motifs (pattern matching), sous-routines, conditions temporelles et recherches en base de données.

Une fois le dialplan maîtrisé, nous ajoutons les fonctionnalités que les utilisateurs attendent réellement d'un PBX — transfert, parcage, prise d'appel, conférence avec ConfBridge et voicemail — pour terminer avec les files d'attente d'appels pour un centre de contact de base.
