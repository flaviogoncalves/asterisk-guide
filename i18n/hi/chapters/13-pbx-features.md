# PBX सुविधाओं का उपयोग

SIP सिस्टम में, अधिकांश फोन सुविधाएं एंडपॉइंट (endpoint) में कार्यान्वित होती हैं। कई प्रकार के SIP फोन और निर्माता मौजूद हैं, और इंटरऑपरेबिलिटी की गारंटी नहीं होती है। Asterisk डेवलपमेंट टीम ने PBX में ही अधिकांश सुविधाओं को लागू करने का अद्भुत काम किया है, जिससे Asterisk लगभग एंडपॉइंट स्वतंत्र हो गया है। हालाँकि, कभी-कभी आप पाएंगे कि एक ही कार्य फोन और Asterisk दोनों द्वारा किया जा रहा है। फोन और PBX का एकीकरण उपयोगिता पर अगली सीमा है और यही वह जगह है जहाँ प्रोप्राइटरी सिस्टम अभी ध्यान केंद्रित कर रहे हैं। इस अध्याय में, आप सीखेंगे कि इन अधिकांश सुविधाओं का उपयोग कैसे किया जाए।

## उद्देश्य

इस अध्याय के अंत तक, आप निम्नलिखित को समझने और उपयोग करने में सक्षम होंगे:

- कॉल पार्किंग (Call Parking)
- कॉल पिकअप (Call Pickup)
- कॉल ट्रांसफर (Call Transfer)
- कॉल कॉन्फ्रेंस (ConfBridge)
- कॉल रिकॉर्डिंग (Call Recording)
- म्यूजिक ऑन होल्ड (Music on hold)

## सुविधाएं कहाँ कार्यान्वित होती हैं

सबसे पहले और सबसे महत्वपूर्ण, यह समझना महत्वपूर्ण है कि PBX सुविधाएं कब निष्पादित हो रही हैं और कब फोन सारा काम कर रहा है। उदाहरण के लिए, आप फोन पर TRANSFER बटन का उपयोग करके या # डायल करके (PBX द्वारा निष्पादित बिना शर्त ट्रांसफर) कॉल ट्रांसफर कर सकते हैं।

## Asterisk द्वारा कार्यान्वित सुविधाएं

ये सुविधाएं Asterisk कोड द्वारा PBX में कार्यान्वित की जाती हैं:

- म्यूजिक ऑन होल्ड
- कॉल पार्किंग
- कॉल पिकअप
- कॉल रिकॉर्डिंग
- ConfBridge कॉन्फ्रेंस रूम
- कॉल ट्रांसफर (ब्लाइंड और कंसल्टेटिव)

## आमतौर पर डायल प्लान द्वारा कार्यान्वित सुविधाएं

इन सुविधाओं को Asterisk डायल प्लान (extensions.conf) में प्रोग्राम करने की आवश्यकता होती है:

- कॉल फॉरवर्ड ऑन बिजी
- कॉल फॉरवर्ड इमीडिएट
- कॉल फॉरवर्ड ऑन अनआंसर्ड
- कॉल फिल्टरिंग (ब्लैकलिस्ट)
- डू नॉट डिस्टर्ब
- रीडायल

## सुविधाओं की कॉन्फ़िगरेशन फ़ाइल

इस अध्याय में प्रस्तुत कुछ सुविधाएं features.conf कॉन्फ़िगरेशन फ़ाइल में कॉन्फ़िगर की गई हैं। इस फ़ाइल को संशोधित करके कुछ सुविधाओं के व्यवहार को बदलना संभव है। हमने नीचे प्रासंगिक अंश शामिल किया है। इस अध्याय के अगले अनुभागों में, हम प्रत्येक सुविधा का वर्णन करेंगे। नमूना फ़ाइल से अंश (Asterisk 22)

![features.conf का `[featuremap]` अनुभाग, डिफ़ॉल्ट DTMF फीचर कोड के साथ](../images/13-pbx-features-fig02.png)

> **[2nd-ed note]** Asterisk 12+ से, कॉल पार्किंग को `features.conf`/`app_features` से हटाकर अपने स्वयं के मॉड्यूल `res_parking` में ले जाया गया है, जिसका कॉन्फ़िगरेशन `res_parking.conf` में है। नीचे दिया गया पार्किंग-लॉट ब्लॉक (parkext, parkpos, context, parkingtime, आदि) `res_parking.conf` में रहता है और Asterisk 22 `res_parking.conf.sample` के सिंटैक्स का उपयोग करके दिखाया गया है। `[featuremap]` अनुभाग (DTMF फीचर कोड, `parkcall` सहित) `features.conf` में रहता है।

पार्किंग-लॉट विकल्प `res_parking.conf` में रहते हैं। `default` नामक एक पार्किंग लॉट हमेशा मौजूद रहता है, भले ही वह कॉन्फ़िगरेशन फ़ाइल में मौजूद न हो। नीचे दिया गया अंश Asterisk 22 `res_parking.conf.sample` से लिया गया है:

```
; res_parking.conf
[default]                       ; Default Parking Lot
parkext => 700                  ; What extension to dial to park. (optional; if
                                ; specified, extensions will be created for parkext and
                                ; the whole range of parkpos)
parkpos => 701-720              ; What range of parking spaces to use - must be numeric.
                                ; Creates these spaces as extensions if parkext is set.
context => parkedcalls          ; Which context parked calls and the default park
                                ; extension are created in
;parkingtime => 45             ; Number of seconds a call can be parked before returning
;comebacktoorigin = yes        ; When a parked call times out, attempt to send it back to
                               ; the peer that parked it (default is yes)
;courtesytone = beep           ; Sound file to play when someone picks up a parked call
;parkedplay = caller           ; Who to play courtesytone to: parked, caller, both (default caller)
;parkedcalltransfers = caller  ; Enable DTMF transfers when picking up a parked call (default no)
;parkedcallreparking = caller  ; Enable DTMF parking when picking up a parked call (default no)
;parkedcallhangup = caller     ; Enable DTMF hangups when picking up a parked call (default no)
;findslot => next              ; 'next' uses the next space after the most recently used one;
                               ; 'first' (default) uses the lowest-numbered space available
;parkedmusicclass = default    ; MOH class to use for the parked channel
```

DTMF फीचर कोड (वन-स्टेप `parkcall` सहित) `features.conf` के `[featuremap]` अनुभाग में रहते हैं:

```
; features.conf
[featuremap]
;blindxfer => #1                ; Blind transfer  (default is #) -- Make sure to set the T and/or t
option in the Dial() or Queue() app call!
;disconnect => *0               ; Disconnect  (default is *) -- Make sure to set the H and/or h option
in the Dial() or Queue() app call!
;automon => *1                  ; One Touch Record a.k.a. Touch Monitor -- Make sure to set the W
and/or w option in the Dial() or Queue() app call!
;atxfer => *2                   ; Attended transfer  -- Make sure to set the T and/or t option in the
Dial() or Queue()  app call!
;parkcall => #72        ; Park call (one step parking)  -- Make sure to set the K and/or k option in
the Dial() app call!
;automixmon => *3               ; One Touch Record a.k.a. Touch MixMonitor -- Make sure to set the X
and/or x option in the Dial() or Queue() app call!
```

## कॉल ट्रांसफर

कॉल ट्रांसफर को फोन, ATA, या स्वयं Asterisk द्वारा कार्यान्वित किया जा सकता है। कॉल ट्रांसफर करने के तरीके को समझने के लिए अपने फोन मैनुअल को देखें। यदि आपका फोन कॉल ट्रांसफर का समर्थन नहीं करता है, तो आप इस कार्य को पूरा करने के लिए Asterisk का उपयोग कर सकते हैं। कॉल ट्रांसफर दो अलग-अलग तरीकों से कार्यान्वित किया जाता है। पहला तरीका ब्लाइंड ट्रांसफर सुविधा का उपयोग करना है: # डायल करें और उसके बाद वह नंबर डायल करें जिसे ट्रांसफर करना है। कभी-कभी आप अपने IP फोन या IP सॉफ्टफोन की ट्रांसफर सुविधा का उपयोग करेंगे। आप features.conf फ़ाइल में blindxfer पैरामीटर को संपादित करके ट्रांसफर कैरेक्टर को बदल सकते हैं। आप features.conf फ़ाइल में atxfer पैरामीटर से पहले ; को हटाकर Asterisk में असिस्टेड ट्रांसफर को सक्षम कर सकते हैं। बातचीत के दौरान, आप *2 दबाएंगे। Asterisk "transfer" कहेगा और आपको डायल टोन देगा। कॉलर को म्यूजिक ऑन होल्ड पर भेज दिया जाता है। गंतव्य व्यक्ति से बात करने और फोन रखने के बाद, सिस्टम कॉलर को गंतव्य से जोड़ देता है।

![कॉल ट्रांसफर: ब्लाइंड ट्रांसफर (कॉल के दौरान # दबाएं) और अटेंडेड ट्रांसफर (कॉल के दौरान *2 दबाएं) के चरण](../images/13-pbx-features-fig03.png)

### कॉन्फ़िगरेशन कार्य सूची

1. यदि फोन SIP आधारित है, तो सुनिश्चित करें कि directmedia विकल्प no के बराबर है या dial() एप्लिकेशन में t या T विकल्प का उपयोग करें।

## कॉल पार्किंग

इस सुविधा का उपयोग कॉल को पार्क करने के लिए किया जाता है। यह तब मदद करता है, उदाहरण के लिए, जब आप अपने कमरे के बाहर फोन कॉल का उत्तर दे रहे हों और आप कॉल को वापस अपने डेस्क पर ट्रांसफर करना चाहते हों। आप इसे किसी एक्सटेंशन में कॉल पार्क करके पूरा कर सकते हैं। एक बार जब आप अपने डेस्क पर पहुँच जाते हैं, तो कॉल को पुनः प्राप्त करने के लिए बस पार्किंग एक्सटेंशन का नंबर डायल करें।

![कॉल पार्किंग: कॉल को पहले खाली स्लॉट (701–720) में पार्क करने के लिए 700 डायल करें; Asterisk स्लॉट की घोषणा करता है, जिसे आप कॉल पुनः प्राप्त करने के लिए किसी भी फोन से डायल करते हैं](../images/13-pbx-features-fig04.png)

डिफ़ॉल्ट रूप से, 700 एक्सटेंशन का उपयोग कॉल को पार्क करने के लिए किया जाता है। बातचीत के बीच में, कॉल को 700 एक्सटेंशन पर ट्रांसफर करने के लिए # दबाएं। अब Asterisk आपके पार्किंग एक्सटेंशन की घोषणा करेगा, जैसे 701 या 702। फोन रख दें, और कॉलर को होल्ड पर रखा जाएगा। अपने डेस्क फोन पर जाएं और कॉल पुनः प्राप्त करने के लिए घोषित पार्किंग एक्सटेंशन डायल करें। यदि कॉलर को लंबे समय तक पार्क किया जाता है, तो टाइमआउट सुविधा ट्रिगर हो जाएगी और मूल रूप से डायल किया गया एक्सटेंशन फिर से बजेगा।

### कॉन्फ़िगरेशन कार्य सूची

कॉल पार्किंग को सक्षम करने के लिए नीचे दिए गए चरणों का पालन करें। चरण 1: पार्किंग लॉट को अपने डायलप्लान से पहुंच योग्य बनाएं (आवश्यक)। डिफ़ॉल्ट पार्किंग लॉट का `context` `parkedcalls` है (`res_parking.conf` में सेट)। उस कॉन्टेक्स्ट को उस कॉन्टेक्स्ट में शामिल करें जहाँ से आपके फोन डायल करते हैं, `extensions.conf` में:

```
include => parkedcalls
```

चरण 2: #700 डायल करके कॉल पार्किंग सुविधा का परीक्षण करें। नोट्स:

- पार्किंग एक्सटेंशन dialplan show CLI कमांड में नहीं दिखाया जाएगा।
- पार्किंग कॉन्फ़िगरेशन फ़ाइल को बदलने के बाद पार्किंग मॉड्यूल को पुनः लोड करना आवश्यक है: `module reload res_parking.so`। features.conf परिवर्तनों के लिए, `module reload features.so`।
- कॉल पार्क करने के लिए, आपको #700 पर ट्रांसफर करने की आवश्यकता है। dial() एप्लिकेशन में t और T विकल्पों को सत्यापित करें।

## कॉल पिकअप

कॉल पिकअप आपको उसी कॉल ग्रुप में किसी सहकर्मी से कॉल कैप्चर करने की अनुमति देता है। यह मदद करेगा, उदाहरण के लिए, उस कॉल को लेने के लिए उठने से बचने में जो आपके कमरे में किसी अन्य व्यक्ति के लिए बज रही है, लेकिन वह वहां मौजूद नहीं है। *8 डायल करके, आप अपने कॉल ग्रुप के भीतर कॉल कैप्चर कर सकते हैं। इस नंबर को

```
features.conf file.
```

![कॉल पिकअप: सदस्य केवल अपने स्वयं के समूह के भीतर कॉल कैप्चर कर सकते हैं; ऑपरेटर (pickupgroup=1,2,3) हर समूह से कॉल पिकअप कर सकता है](../images/13-pbx-features-fig05.png)

### कॉन्फ़िगरेशन कार्य सूची

कॉल पिकअप सुविधा को कॉन्फ़िगर करने के लिए नीचे दिए गए चरणों का पालन करें। चरण 1: अपने एक्सटेंशन के लिए एक कॉल ग्रुप कॉन्फ़िगर करें। यह चैनल कॉन्फ़िगरेशन फ़ाइल (pjsip.conf, iax.conf, chan_dahdi.conf) में किया जाता है। PJSIP एंडपॉइंट्स के लिए, `pjsip.conf` के एंडपॉइंट अनुभाग में `call_group` और `pickup_group` सेट करें (pjsip.conf snake_case विकल्प नामों का उपयोग करता है)। यह कार्य आवश्यक है।

PJSIP (pjsip.conf) के लिए:
```
[4x00]
type=endpoint
call_group=1
pickup_group=1,2
```


चरण 2: कॉल-पिकअप फीचर नंबर बदलें (वैकल्पिक)।

```
pickupexten=*8; Configures the call pickup extension
```

## कॉन्फ्रेंस (कॉल कॉन्फ्रेंस)

Asterisk पर कॉन्फ्रेंस को कार्यान्वित करने के अलग-अलग तरीके हैं। पहला विकल्प फोन की थ्री-वे कॉन्फ्रेंस क्षमता का उपयोग करना है। फोन में इस सुविधा का उपयोग करके आपको सर्वर में किसी समर्थन की आवश्यकता नहीं होती है। हालाँकि, जब आप 3 से अधिक लोगों के साथ कॉन्फ्रेंस करना चाहते हैं, तो आपको एक कॉन्फ्रेंस रूम चलाना चाहिए। Asterisk का आधुनिक कॉन्फ्रेंस एप्लिकेशन ConfBridge (`app_confbridge`) है।

ConfBridge HD वॉयस कॉन्फ्रेंस और वीडियो कॉन्फ्रेंसिंग का समर्थन करता है। वीडियोकॉन्फ्रेंस के लिए कुछ सीमाएं हैं जैसे कोई ट्रांसकोडिंग नहीं — सभी प्रतिभागियों को एक ही कोडेक और प्रोफाइल का उपयोग करना होगा। वीडियोकॉन्फ्रेंस 'फॉलो-द-टॉकर' मोड का उपयोग करती है, जो बोलने वाले अंतिम व्यक्ति की छवि प्रदर्शित करती है। आप ConfBridge में आसानी से नए DTMF मेनू कॉन्फ़िगर कर सकते हैं।

> **[2nd-ed note]** MeetMe (`app_meetme`) को **Asterisk 19 में डेप्रिकेट (deprecated)** कर दिया गया था और इसे Asterisk 21 में हटाने के लिए निर्धारित किया गया था, लेकिन वह निष्कासन रोक दिया गया था (ViciDial प्रोजेक्ट के अनुरोध पर)। मॉड्यूल स्रोत अभी भी Asterisk 22 के साथ आता है, लेकिन इसके लिए DAHDI की आवश्यकता होती है और यह **डिफ़ॉल्ट Asterisk 22 बिल्ड द्वारा निर्मित नहीं होता है** — एक मानक इंस्टॉलेशन में कोई `app_meetme.so` नहीं होता है और `MeetMe()` एप्लिकेशन अनुपलब्ध है। सभी नए कॉन्फ्रेंस रूम परिनियोजन को ConfBridge का उपयोग करना चाहिए। नीचे दिया गया MeetMe अनुभाग केवल ऐतिहासिक संदर्भ के लिए रखा गया है।

### Confbridge

कॉन्फ्रेंस रूम शुरू करने के लिए, सिंटैक्स नीचे सूचीबद्ध है।

```
ConfBridge(conference,bridge_profile,user_profile,menu)
```

कमांड का पूरा विवरण प्राप्त करने के लिए आप core show application confbridge का उपयोग कर सकते हैं।

![`core show application confbridge` का आउटपुट, जिसमें सारांश, सिंटैक्स, और bridge_profile, user_profile, और मेनू तर्क दिखाए गए हैं](../images/13-pbx-features-fig06.png)

जैसा कि आप ऊपर देख सकते हैं, तीन महत्वपूर्ण अनुभाग हैं: Bridge_profile: आप confbridge.conf फ़ाइल में प्रोफाइल को परिभाषित करते हैं। वहाँ आप प्रतिभागियों की अधिकतम संख्या, रिकॉर्डिंग, video_mode और कई अन्य ब्रिज पैरामीटर चुन सकते हैं।

यहाँ पूरी उदाहरण फ़ाइल को पुन: प्रस्तुत करने का कोई मतलब नहीं है, इसलिए मुझे confbridge.conf फ़ाइल में bridge_profile को कॉन्फ़िगर करने का एक सरल उदाहरण देने दें।

```
[default_bridge]
type=bridge
max_members=10
record_conference=yes
```

User_profile: यहाँ आप उन विकल्पों को परिभाषित करते हैं जो प्रति उपयोगकर्ता विशिष्ट हैं जैसे कि उपयोगकर्ता प्रशासक है या नहीं। म्यूजिक ऑन होल्ड और कई अन्य विकल्प प्रति उपयोगकर्ता सेट किए जा सकते हैं। उदाहरण:

```
[admin_user]
type=user
admin=yes
```

Menu: मेनू अनुभाग में आप एप्लिकेशन के लिए अपना कीबोर्ड मैपिंग परिभाषित कर सकते हैं जहाँ म्यूट और अनम्यूट को टॉगल करना है। विकल्पों को देखने के लिए confbridge.conf फ़ाइल देखें। उदाहरण:

```
[my_menu]
type=menu
*=playback_and_continue
1=toggle_mute
2=decrease_listening_volume
3=increase_listening_volume
4=decrease_talking_volume
5=increase_talking_volume
6=leave_conference
```

#### Confbridge फ़ंक्शंस

कॉन्फ्रेंस ब्रिज विकल्पों को CONFBRIDGE() फ़ंक्शन का उपयोग करके डायल प्लान में गतिशील रूप से पारित किया जा सकता है। नीचे दिए गए उदाहरण देखें:

```
exten => 1,1,Answer()
exten => 1,n,Set(CONFBRIDGE(user,template)=default_user)
exten => 1,n,Set(CONFBRIDGE(user,admin)=yes)
exten => 1,n,Set(CONFBRIDGE(user,marked)=yes)
exten => 1,n,ConfBridge(sales)
```

## Meetme (लिगेसी — डेप्रिकेटेड, डिफ़ॉल्ट रूप से Asterisk 22 में निर्मित नहीं)

> **[2nd-ed note]** `app_meetme` को Asterisk 19 में डेप्रिकेट कर दिया गया था। Asterisk 21 में इसकी नियोजित निष्कासन को रोक दिया गया था, इसलिए स्रोत अभी भी Asterisk 22 में आता है, लेकिन मॉड्यूल डिफ़ॉल्ट Asterisk 22 बिल्ड द्वारा निर्मित नहीं है (यह DAHDI पर निर्भर करता है)। इसलिए एक मानक Asterisk 22 इंस्टॉलेशन में कोई `MeetMe()` एप्लिकेशन नहीं है। नीचे दी गई सामग्री ऐतिहासिक संदर्भ के लिए और पुराने सिस्टम से अपग्रेड करने वाले पाठकों के लिए रखी गई है। **नए इंस्टॉलेशन के लिए, ConfBridge का उपयोग करें (ऊपर देखें)।** ConfBridge के लिए DAHDI निर्भरता और `dahdi_dummy` मॉड्यूल की आवश्यकता नहीं है।

वैकल्पिक रूप से, पुराने Asterisk संस्करणों में आप meetme() एप्लिकेशन का उपयोग कर सकते थे। Meetme एक कॉन्फ्रेंस ब्रिज है जिसका उपयोग करना बहुत आसान है। याद रखें, meetme को Asterisk 19 में डेप्रिकेट कर दिया गया था और सिंक्रोनाइज़ेशन के लिए DAHDI मॉड्यूल पर निर्भर करता है।

![MeetMe कॉन्फ्रेंस प्रकार — सिंगल-स्पीकर, पासवर्ड-संरक्षित, और गतिशील कॉन्फ्रेंस — सभी को Zaptel/DAHDI टाइमिंग स्रोत की आवश्यकता होती है](../images/13-pbx-features-fig07.png)

### meetme() एप्लिकेशन

meetme show CLI कमांड का उपयोग करके, आप उपरोक्त विवरण प्राप्त कर सकते हैं। meetme का उपयोग करने के लिए, आपको DAHDI ड्राइवरों को संकलित करने और कम से कम एक DAHDI कर्नेल मॉड्यूल लोड करने की आवश्यकता है। यदि आपके पास कम से कम एक DAHDI कार्ड स्थापित नहीं है, तो टाइमिंग स्रोत प्रदान करने के लिए dahdi_dummy कर्नेल मॉड्यूल लोड करें। विवरण:

![MeetMe() एप्लिकेशन: सिंटैक्स `MeetMe([confno][,[options][,pin]])` और इसके मुख्य विकल्प फ्लैग](../images/13-pbx-features-fig08.png)

meetme() एप्लिकेशन उपयोगकर्ता को एक निर्दिष्ट meetme कॉन्फ्रेंस में ले जाता है। यदि कॉन्फ्रेंस नंबर छोड़ दिया जाता है, तो उपयोगकर्ता को एक दर्ज करने के लिए कहा जाएगा। उपयोगकर्ता फोन रखकर कॉन्फ्रेंस छोड़ सकता है या—यदि p विकल्प निर्दिष्ट है—तो # दबाकर। कृपया ध्यान दें: कॉन्फ्रेंसिंग को ठीक से संचालित करने के लिए DAHDI कर्नेल मॉड्यूल और कम से कम एक हार्डवेयर ड्राइवर (या dahdi_dummy) मौजूद होना चाहिए। इसके अलावा, i और r विकल्पों के काम करने के लिए chan_dahdi चैनल ड्राइवर लोड होना चाहिए।

> **[2nd-ed note]** नीचे दी गई विकल्प-फ्लैग और एडमिन-कमांड सूचियां लिगेसी `app_meetme` इंटरफ़ेस का वर्णन करती हैं और ऐतिहासिक `MeetMe()`/`MeetMeAdmin()` दस्तावेज़ीकरण को दर्शाती हैं। चूंकि `app_meetme` डिफ़ॉल्ट Asterisk 22 इंस्टॉलेशन द्वारा निर्मित नहीं है, इसलिए इन फ्लैग्स की पुष्टि चल रहे Asterisk 22 सिस्टम के विरुद्ध नहीं की जा सकती; उन्हें पुराने परिनियोजन को बनाए रखने वाले पाठकों के लिए पुन: प्रस्तुत किया गया है। Asterisk 22 पर, ConfBridge और `CONFBRIDGE()` डायलप्लान फ़ंक्शन का उपयोग करें।

विकल्प स्ट्रिंग में निम्नलिखित वर्णों में से शून्य या एक या अधिक हो सकते हैं:

- 'a' -- एडमिन मोड सेट करता है
- 'A' -- मार्क्ड मोड सेट करता है
- 'b' – ${MEETME_AGI_BACKGROUND} में निर्दिष्ट AGI स्क्रिप्ट चलाता है। डिफ़ॉल्ट: conf-background.agi (नोट: यह उसी कॉन्फ्रेंस में गैर-DAHDI चैनलों के साथ काम नहीं करता है)
- 'c' -- कॉन्फ्रेंस में शामिल होने पर उपयोगकर्ता(ओं) की संख्या की घोषणा करता है
- 'd' -- गतिशील रूप से कॉन्फ्रेंस जोड़ता है
- 'D' -- गतिशील रूप से कॉन्फ्रेंस जोड़ता है, PIN के लिए संकेत देता है
- 'e' -- एक खाली कॉन्फ्रेंस चुनता है
- 'E' -- एक खाली पिनलेस कॉन्फ्रेंस चुनता है
- 'i' -- समीक्षा के साथ शामिल होने/छोड़ने वाले उपयोगकर्ता की घोषणा करता है
- 'I' -- बिना समीक्षा के शामिल होने/छोड़ने वाले उपयोगकर्ता की घोषणा करता है
- 'l' -- केवल सुनने का मोड सेट करता है (केवल सुनें, कोई बात नहीं)
- 'm' -- शुरू में म्यूट सेट करता है
- 'M' -- जब कॉन्फ्रेंस में एक ही कॉलर हो तो म्यूजिक ऑन होल्ड सक्षम करता है
- 'o' -- टॉकर ऑप्टिमाइज़ेशन सेट करता है, जो उन बोलने वालों को म्यूट के रूप में मानता है जो नहीं बोल रहे हैं, जिसका अर्थ है (a) ट्रांसमिशन पर कोई एन्कोड नहीं किया जाता है और (b) प्राप्त ऑडियो जो बोलने के रूप में पंजीकृत नहीं है उसे छोड़ दिया जाता है, जिससे पृष्ठभूमि शोर में कोई वृद्धि नहीं होती है
- 'p' -- उपयोगकर्ताओं को '#' दबाकर कॉन्फ्रेंस से बाहर निकलने की अनुमति देता है
- 'P' -- हमेशा पिन के लिए संकेत देता है भले ही वह निर्दिष्ट हो
- 'q' -- शांत मोड (प्रवेश/निकास ध्वनियां न बजाएं)
- 'r' -- कॉन्फ्रेंस रिकॉर्ड करता है (${MEETME_RECORDINGFILE} के रूप में रिकॉर्ड करता है, ${MEETME_RECORDINGFORMAT} प्रारूप का उपयोग करके)। डिफ़ॉल्ट फ़ाइल नाम meetme-conf-rec-${CONFNO}-${UNIQUEID} है और डिफ़ॉल्ट प्रारूप wav है।
- 's' -- जब '*' प्राप्त होता है तो मेनू (उपयोगकर्ता या एडमिन) प्रस्तुत करता है ('send' to menu)
- 't' -- केवल बात करने का मोड सेट करता है। (केवल बात करें, कोई सुनना नहीं)
- 'T' -- टॉकर डिटेक्शन सेट करता है (मैनेजर इंटरफ़ेस और meetme सूची में भेजा गया)
- 'w[(<secs>)]' -- मार्क्ड उपयोगकर्ता के कॉन्फ्रेंस में प्रवेश करने तक प्रतीक्षा करता है
- 'x' -- जब अंतिम मार्क्ड उपयोगकर्ता बाहर निकलता है तो कॉन्फ्रेंस बंद कर देता है
- 'X' -- उपयोगकर्ता को एक वैध एकल अंक एक्सटेंशन ${MEETME_EXIT_CONTEXT} या वर्तमान कॉन्टेक्स्ट (यदि वह वेरिएबल परिभाषित नहीं है) दर्ज करके कॉन्फ्रेंस से बाहर निकलने की अनुमति देता है।
- '1' -- जब पहला व्यक्ति प्रवेश करता है तो संदेश नहीं बजाता है

### Meetme कॉन्फ़िगरेशन फ़ाइल

इस फ़ाइल का उपयोग एप्लिकेशन meetme को कॉन्फ़िगर करने के लिए किया जाता है। उदाहरण के लिए:

```
;
; Configuration file for MeetMe simple conference rooms for Asterisk of course.
;
; This configuration file is read every time you call app meetme()
[general]
;audiobuffers=32        ; The number of 20ms audio buffers to be used
                        ; when feeding audio frames from non-DAHDI channels
                        ; into the conference; larger numbers will allow
                        ; for the conference to 'de-jitter' audio that arrives
                        ; at different timing than the conference's timing
                        ; source, but can also allow for latency in hearing
                        ; the audio from the speaker. Minimum value is 2,
                        ; maximum value is 32.
;
[rooms]
;
; Usage is conf => confno[,pin][,adminpin]
;
conf=>9000
conf=>9001,123456
```

meetme.conf फ़ाइल में परिवर्तनों को Asterisk द्वारा देखने के लिए reload या restart का उपयोग करना आवश्यक नहीं है।

### Meetme-संबंधित एप्लिकेशन

meetme() एप्लिकेशन में दो अन्य समर्थन एप्लिकेशन हैं।

```
MeetMeCount(confno[|var])
```

यह कॉन्फ्रेंस में उपयोगकर्ताओं की संख्या बजाता है। यदि कोई वेरिएबल निर्दिष्ट है, तो यह संदेश नहीं बजाता है लेकिन उपयोगकर्ताओं की संख्या को उस पर सेट कर देता है।

```
MeetMeAdmin(confno,command,[user]):
```

कॉन्फ्रेंस के लिए एडमिन कमांड चलाएं:

- 'e' -- शामिल होने वाले अंतिम उपयोगकर्ता को बाहर निकालता है
- 'k' -- एक उपयोगकर्ता को कॉन्फ्रेंस से बाहर निकालता है
- 'K' -- सभी उपयोगकर्ताओं को कॉन्फ्रेंस से बाहर निकालता है
- 'l' -- कॉन्फ्रेंस को अनलॉक करता है
- 'L' -- कॉन्फ्रेंस को लॉक करता है
- 'm' -- एक उपयोगकर्ता को अनम्यूट करता है
- 'M' -- एक उपयोगकर्ता को म्यूट करता है
- 'n' -- कॉन्फ्रेंस में सभी उपयोगकर्ताओं को अनम्यूट करता है
- 'N' -- कॉन्फ्रेंस में सभी गैर-एडमिन उपयोगकर्ताओं को म्यूट करता है
- 'r' -- एक उपयोगकर्ता की वॉल्यूम सेटिंग्स को रीसेट करता है
- 'R' -- सभी उपयोगकर्ताओं की वॉल्यूम सेटिंग्स को रीसेट करता है
- 's' -- पूरी कॉन्फ्रेंस की बोलने की वॉल्यूम कम करता है
- 'S' -- पूरी कॉन्फ्रेंस की बोलने की वॉल्यूम बढ़ाता है
- 't' -- एक उपयोगकर्ता की बात करने की वॉल्यूम कम करता है
- 'T' -- सभी उपयोगकर्ताओं की बात करने की वॉल्यूम कम करता है
- 'u' -- एक उपयोगकर्ता की सुनने की वॉल्यूम कम करता है
- 'U' -- सभी उपयोगकर्ताओं की सुनने की वॉल्यूम कम करता है
- 'v' -- पूरी कॉन्फ्रेंस की सुनने की वॉल्यूम कम करता है
- 'V' -- पूरी कॉन्फ्रेंस की सुनने की वॉल्यूम बढ़ाता है

### Meetme कॉन्फ़िगरेशन कार्य सूची

meetme कॉन्फ्रेंसिंग एप्लिकेशन को कॉन्फ़िगर करने के लिए नीचे दिए गए चरणों का पालन करें। चरण 1: Meetme रूम के लिए एक्सटेंशन चुनें (आवश्यक) चरण 2: पासवर्ड कॉन्फ़िगर करने के लिए meetme.conf फ़ाइल संपादित करें (वैकल्पिक)

### उदाहरण

उदाहरण #1: सरल meetme रूम 1. extensions.conf फ़ाइल में, कॉन्फ्रेंस रूम 101 बनाएं

```
exten=>500,1,MeetMe(101,,123456)
```

2. meetme.conf फ़ाइल में, रूम 101 के लिए पासवर्ड स्थापित करें। महत्वपूर्ण नोट: meetme() एप्लिकेशन को काम करने के लिए एक टाइमर की आवश्यकता होती है। यदि आपके पास digium हार्डवेयर स्थापित और कॉन्फ़िगर नहीं है, तो टाइमिंग स्रोत के रूप में dahdi_dummy का उपयोग करें।

## कॉल रिकॉर्डिंग

Asterisk में कॉल रिकॉर्ड करने के कई तरीके हैं। आप कॉल को आसानी से रिकॉर्ड करने के लिए mixmonitor() एप्लिकेशन का उपयोग कर सकते हैं।

### mixmonitor एप्लिकेशन का उपयोग करना

mixmonitor एप्लिकेशन वर्तमान चैनल में ऑडियो को निर्दिष्ट फ़ाइल में रिकॉर्ड करता है। यदि फ़ाइल नाम एक पूर्ण पथ है, तो यह उस पथ का उपयोग करता है। अन्यथा, यह asterisk.conf से कॉन्फ़िगर की गई मॉनिटरिंग निर्देशिका में फ़ाइल बनाता है।

![MixMonitor() एप्लिकेशन: एक चैनल के ऑडियो को एक फ़ाइल में रिकॉर्ड और मिक्स करता है, जिसमें अपेंड, केवल-ब्रिज्ड, और वॉल्यूम समायोजन के विकल्प हैं](../images/13-pbx-features-fig09.png)

### Mixmonitor()

कॉल रिकॉर्ड करें और रिकॉर्डिंग के दौरान ऑडियो मिक्स करें [विवरण] MixMonitor(<file>.<ext>[|<options>[|<command>]]) वर्तमान चैनल पर ऑडियो को निर्दिष्ट फ़ाइल में रिकॉर्ड करता है। विकल्प: a- फ़ाइल को ओवरराइट करने के बजाय उसमें जोड़ें (Append)। b- केवल तभी फ़ाइल में ऑडियो सहेजें जब चैनल ब्रिज्ड हो। नोट: इसमें कॉन्फ्रेंस शामिल नहीं हैं। v(<x>) - सुनी गई वॉल्यूम को <x> के कारक से समायोजित करें। V(<x>) - बोली गई वॉल्यूम को <x> के कारक से समायोजित करें। W(<x>) - सुनी गई और बोली गई दोनों वॉल्यूम को समायोजित करें। वैध विकल्प:

- a - फ़ाइल को ओवरराइट करने के बजाय उसमें जोड़ता है।
- b - केवल तभी ऑडियो को फ़ाइल में सहेजता है जब चैनल ब्रिज्ड हो।
- नोट: इसमें कॉन्फ्रेंस शामिल नहीं हैं।
- v(<x>) - सुनी गई वॉल्यूम को <x> के कारक से समायोजित करता है (-4 से 4 तक)।
- V(<x>) - बोली गई वॉल्यूम को <x> के कारक से समायोजित करता है (-4 से 4 तक)।
- W(<x>) - सुनी गई और बोली गई दोनों वॉल्यूम को <x> के कारक से समायोजित करता है (-4 से 4 तक)।
- रिकॉर्डिंग समाप्त होने पर <command> निष्पादित किया जाएगा। ^{X} से मेल खाने वाली कोई भी स्ट्रिंग ${X} में अनएस्केप हो जाएगी और उस समय सभी वेरिएबल्स का मूल्यांकन किया जाएगा। वेरिएबल MIXMONITOR_FILENAME में रिकॉर्ड करने के लिए उपयोग किया गया फ़ाइल नाम होगा।

एक दिलचस्प संसाधन automon है, जो आपको तुरंत रिकॉर्डिंग शुरू करने के लिए बस *1 डायल करने की अनुमति देता है। उदाहरण:

```
exten=>_4XXX,1,Set(DYNAMIC_FEATURES=automon)
exten=>_4XXX,2,Dial(PJSIP/${EXTEN},20,jtTwW);wW enables the recording.
```

ऑडियो चैनल इनकमिंग (IN) और आउटगोइंग (OUT) हैं और /var/spool/asterisk/monitor निर्देशिका में दो अलग-अलग फ़ाइलों में विभाजित हैं। दोनों फ़ाइलों को sox एप्लिकेशन का उपयोग करके मिक्स किया जा सकता है।

```
debian#soxmix *in.wav *out.wav output.wav
```

यदि आप Dial() एप्लिकेशन से पहले Set() का उपयोग नहीं करना चाहते हैं, तो आप इसे globals अनुभाग में सेट कर सकते हैं:

```
[globals]
DYNAMIC_FEATURES=>automon
```

### म्यूजिक ऑन होल्ड

म्यूजिक ऑन होल्ड (MOH) संस्करण 1.0, 1.2, और 1.4 के बीच कई बार बदल गया है। नवीनतम संस्करण में, MOH डिफ़ॉल्ट रूप से "FILE-BASED" है। दूसरे शब्दों में, Asterisk g729, alaw, ulaw, और gsm जैसे प्रारूपों में MOH फ़ाइलें प्रदान करेगा। इस प्रकार, संगीत को चैनल पर भेजने से पहले ट्रांसकोड करना आवश्यक नहीं है। यह प्रोसेसर का समय बचाता है, जो उत्पादन प्रणालियों के साथ काम करने वालों के लिए एक स्वागत योग्य संशोधन है। पुराने संस्करणों में, MOH आमतौर पर MP3 द्वारा प्रदान किया जाता था (इसे अभी भी उस तरह से कॉन्फ़िगर किया जा सकता है)। MP3 का उपयोग करके MOH प्रदान करना Asterisk को ट्रांसकोड करने के लिए बाध्य करता है, जिससे प्रक्रिया में मूल्यवान CPU शक्ति खर्च होती है। नई कॉन्फ़िगरेशन फ़ाइल नीचे दिखाई गई है। ध्यान दें कि डिफ़ॉल्ट क्लास अब नेटिव फ़ाइल प्रारूप mode=files का उपयोग करती है। अन्य सभी मोड कमेंट किए गए हैं। प्रत्येक अनुभाग एक क्लास है। इस बिंदु पर एकमात्र अनकमेंटेड क्लास default है। यदि आप अलग-अलग फ़ाइलों के लिए अलग-अलग क्लास रखना चाहते हैं, तो आपको नए अनुभाग (क्लास) बनाने होंगे।

![musiconhold.conf नमूना कॉन्फ़िगरेशन, वैध MOH मोड (quietmp3, mp3, custom, files, …) को सूचीबद्ध करता है](../images/13-pbx-features-fig10.png)


### MOH कॉन्फ़िगरेशन कार्य

अब, म्यूजिक ऑन होल्ड का उपयोग करने के लिए, चैनल कॉन्फ़िगरेशन फ़ाइलों (chan_dahdi.conf, pjsip.conf, iax.conf, आदि) में MOH क्लास सेट करें। PJSIP एंडपॉइंट्स के लिए, `pjsip.conf` के एंडपॉइंट अनुभाग में `moh_suggest` सेट करें (लिगेसी `musicclass` विकल्प नाम chan_dahdi और अन्य चैनल ड्राइवरों पर लागू होता है, PJSIP पर नहीं)। इंस्टॉल की गई फ्रीप्ले ट्यून्स अब wav प्रारूप में हैं। इंस्टॉलेशन के समय, आप उपलब्ध MOH फ़ाइल प्रारूपों का चयन कर सकते हैं (make menuselect का उपयोग करके)। यदि आप नई MOH फ़ाइलें जोड़ना चाहते हैं, तो आपको उन्हें आवश्यक प्रारूपों में प्रदान करना होगा। उदाहरण के लिए:

```
In /etc/asterisk/chan_dahdi.conf, add the line:
[channels]
musiconhold=default
Edit the file /etc/asterisk/musiconhold.conf
[default]
mode=files
directory=/var/lib/asterisk/moh
```

डायल प्लान में, आप निम्नलिखित उदाहरण का उपयोग करके MOH सुन सकते हैं:

```
Exten=>100,1,SetMusicOnHold(default)
Exten=>100,2,Dial(DAHDI/2)
```

MOH का परीक्षण करने के लिए फ़ाइल extensions.conf को कॉन्फ़िगर करने के लिए:

```
[local]
exten => 6601,1,WaitMusicOnHold(30)
```

## एप्लिकेशन मैप्स

एप्लिकेशन मैप्स आपको features.conf फ़ाइल के `[applicationmap]` अनुभाग का उपयोग करके नई सुविधाएं जोड़ने की अनुमति देते हैं। मान लीजिए कि आपको कॉल सेंटर में कॉल का उत्तर देते समय ग्राहक के प्रकार की पहचान करने की आवश्यकता है। आप प्रत्येक ग्राहक प्रकार के लिए एक एप्लिकेशन मैप बना सकते हैं, जो प्रति प्रकार उत्तर दिए गए ग्राहकों की संख्या की गणना कर सके।
